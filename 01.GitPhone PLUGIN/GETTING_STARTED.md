# Getting Started with GitPhone Emperor

Welcome! This guide helps you navigate the Git & GitHub mastery course, understand the learning method, and use every resource effectively — entirely from your phone. No laptop required.

---

## 1. The Course at a Glance

GitPhone Emperor is a complete, step‑by‑step curriculum that turns you from a complete beginner into a confident Git and GitHub user. It’s structured into **four modules**, each containing multiple focused lessons.

- **Module 1 – Git Foundations:** Tracking your own work, committing, viewing history, ignoring files.
- **Module 2 – Branching & Collaboration:** Working on multiple features, merging, resolving conflicts.
- **Module 3 – GitHub & Open‑Source Workflow:** Cloning, forking, pull requests, code reviews.
- **Module 4 – Professional Git Craft:** Rebasing, cherry‑picking, reflog, hooks, and advanced debugging.

For the full lesson list, open `02.GitPhone SYLLABUS/GitPhone_Curriculum.md`.

---

## 2. Folder Structure

Every folder in the repository has a specific job. Knowing this will help you move fast.

- **`01.GitPhone PLUGIN/`** – Setup and getting‑started files. You are reading one right now.
- **`02.GitPhone SYLLABUS/`** – The full course outline and the AI mentor prompt.
- **`03.GitPhone COURSE/`** – All lessons, grouped by module. Each lesson is a `.md` file you open, read, and practice.
- **`04.GitPhone NOTES/`** – (Optional but recommended) One‑page revision sheets you write yourself after each lesson.
- **`05.GitPhone PRACTISE/`** – Practice sets that mix everything from a module into real‑world tasks.

Take a moment now to open Acode and explore these folders. Familiarity builds speed.

---

## 3. The Learning Method – Two Bricks at a Time

The entire course uses the **two bricks** method. Every lesson introduces **only one new concept** or **two very small ones** that belong together. Never more.

Why? Because building true skill is like building a wall:
- You place one brick, let the mortar set, then add another.
- Rushing leads to cracks. Patience builds a fortress.

After each lesson, you get a **tiny practice task**. You type every command yourself in Termux, observe the output, and only then move on. No copy‑paste, no skipping.

---

## 4. Daily Workflow

1. **Open the lesson** in Acode (inside `03.GitPhone COURSE/` → the current module).
2. **Read both bricks** slowly. Understand the “why” before the “how”.
3. **Open Termux** and perform the practice task exactly as described.
4. **Write a short note** (1–3 lines) in your `04.GitPhone NOTES/` file for that lesson. This cements the concept.
5. **Commit and push** your progress. Every lesson is a milestone you save to GitHub.

If you ever get stuck, re‑read the lesson, run the commands again, or ask your AI mentor (see Section 5).

---

## 5. Using the AI Mentor

Inside `02.GitPhone SYLLABUS/`, you’ll find a file called `GitPhone_AI_Prompt.md`. That file contains a complete prompt that turns any modern AI assistant into your personal Git mentor.

How to activate it:
- Open the file in Acode and copy its entire content.
- Paste it as the **first message** in a new conversation with an AI (DeepSeek, ChatGPT, etc.).
- The AI will then guide you through the syllabus using the same two‑bricks method.

No configuration needed — it’s plug‑and‑play.
