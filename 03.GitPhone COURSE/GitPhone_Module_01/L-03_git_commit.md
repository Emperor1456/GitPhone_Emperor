📝 Lesson 3 – git commit

What it does:
git commit saves the staged changes as a permanent snapshot.
Every commit gets a unique hash and a message.
Commit often — each one is a restore point you can return to.

Practice
Run these in the same git-practice sandbox:

🚀 Commands:
· git commit -m "First commit"
· git log --oneline
· git status
---------------------------------

You’ll see the commit message, a short hash, and “nothing to commit” — meaning the snapshot is safely stored.
