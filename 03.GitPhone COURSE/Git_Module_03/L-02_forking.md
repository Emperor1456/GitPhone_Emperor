📝 Lesson 2 – Forking on GitHub

📖 What it does:
→ Forking creates your own copy of someone else's repository on GitHub.
→ You can make changes without affecting the original project.
→ Forks are the starting point for contributing to open source.

🛠️ Practice:
Fork a repository from GitHub's web interface.

🚀 Commands:
→ Open browser, go to https://github.com/octocat/Spoon-Knife
→ Tap the "Fork" button (top right).
→ Choose your account as the destination.
→ After forking, you have a copy at https://github.com/<your-username>/Spoon-Knife.
$ cd /storage/emulated/0/Download
→ Go to workspace.
$ git clone https://github.com/<your-username>/Spoon-Knife.git
→ Clone your fork.
$ cd Spoon-Knife
→ Enter the fork.
$ git remote -v
→ origin points to your fork.
-------------------------------------

Forking is a GitHub feature, not a Git command.
